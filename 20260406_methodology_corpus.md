> 작성일: 2026년 4월  
> 상태: 수집·전처리 완료 / 분석 단계 진입 전

---

## 1. 방법론 명칭

**Corpus-Assisted Hybrid Approach (코퍼스 보조 혼합 접근법)**

계량적 기법(corpus-assisted)과 질적 기법(hybrid)을 결합한 혼합 방법론(mixed methods)이다.

- **이론적 근거:** Tognini-Bonelli(2001), Baker et al.(2008)
- **설계 유형:** 설명적 순차 혼합 설계(explanatory sequential mixed methods design)
  - 양적 분석이 "큰 그림"을 보여주고, 질적 분석이 "왜 그런지"를 설명

### spaCy 채택 근거

NLTK의 어간 추출(stemming)은 단어를 사전에 존재하지 않는 형태로 잘라내는 반면, spaCy의 표제어 추출(lemmatization)은 사전형으로 정확하게 변환한다. 학술 전문 용어(`hardware`, `authenticity`, `provenance` 등)의 원형을 정확하게 보존해야 하는 본 연구의 특성상 spaCy가 적합하다.

| | NLTK (stemming) | spaCy (lemmatization) |
|--|----------------|----------------------|
| archives | archiv ❌ | archive ✅ |
| studies | studi ❌ | study ✅ |
| running | run ✅ | run ✅ |

| 2 | `02_tokenization.py` | 토큰화, 표제어 추출, 품사 필터링(명사·동사·형용사·고유명사) | ✅ 완료 |

### 불용어 제거 건너뜀 근거

단어 빈도 분석(`frequency_analysis.py`, 9,964개 고유 단어)을 통해 상위 200개 단어를 검토한 결과, 다음과 같은 이유로 불용어 제거를 보류하였다.

1. **분석 목적과의 불일치:** 본 연구의 핵심 목적은 하드웨어·물질성 관련 용어의 **구조적 부재** 실증이다. 범용 형용사·동사 제거는 키워드 빈도 분석 및 공기어 분석에 실질적 영향을 미치지 않는다.
2. **복합어 보존:** `archival material`, `trusted computing`, `hardware layer` 같은 복합 표현을 보존하기 위해 명사류 불용어 처리를 보류하였다.
3. **단계별 적용:** 토픽 모델링(LDA/BERTopic) 단계에서 필요 시 해당 분석에 한정하여 불용어를 적용할 예정이다.

## 5. 4단계 분석 구조 (예정)

| 단계 | 방법 | 내용 | 도구 |
|------|------|------|------|
| 1단계 | 키워드 빈도 분석 | Field별 키워드 출현 빈도, 연도별 시계열 추적 | AntConc, Python |
| 2단계 | 변화점 탐지 (change point detection) | 사전 시기 구분 없이 데이터에서 담론 전환점 도출 | Python |
| 3단계 | 공기어(co-occurrence) 분석 | 핵심 키워드 주변 맥락 어휘 패턴 분석 | AntConc |
| 4단계 | 질적 맥락 코딩 | 물질성 범주별 코딩, 코더 간 신뢰도 확보 | 수동 |

### 핵심 설계 원칙

- **시기를 미리 나누지 않는다.** 변화점 탐지 결과로 사후 시기를 구분한다. 이는 "왜 그 연도에서 잘랐는가"라는 자의성 비판을 차단하기 위한 방법론적 선택이다.
- **문헌량이 적은 최근 시기(2020년 이후)는 질적 분석으로 보완한다.**

> **주의:** 4단계 분석 구조는 현재 확정된 방향이나, 세부 실행은 분석 단계 진입 후 확정한다. 분석 결과(예: 특정 시기 담론 전환 비율 등)는 실행 전까지 방법론 문서에 포함하지 않는다.
